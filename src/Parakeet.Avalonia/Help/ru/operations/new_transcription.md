# Рабочий процесс новой транскрипции

Используйте этот рабочий процесс для транскрипции одного аудиофайла.

## Предварительные требования

- Все файлы моделей должны быть загружены. Карточка **Статус модели** должна отображать `All N model file(s) present ✓`. См. [Загрузка моделей](../first_steps/downloading_models.md).

## Поддерживаемые форматы

### Аудио

`.wav`, `.mp3`, `.flac`, `.m4a`, `.ogg`, `.aac`, `.wma`, `.aiff`, `.aif`, `.webm`

### Видео

`.mp4`, `.mov`, `.mkv`, `.avi`, `.wmv`, `.flv`, `.ts`, `.mts`, `.m2ts`, `.3gp`

Видеофайлы декодируются с помощью FFmpeg. Если видеофайл содержит **несколько аудиопотоков** (например, несколько языков или дорожки с комментариями), для каждого потока автоматически создаётся отдельное задание транскрипции.

## Шаги

### 1. Откройте форму новой транскрипции

Нажмите `New Transcription` на главном экране или перейдите в `File > New Transcription`.

### 2. Выберите медиафайл

Нажмите `Browse…` рядом с полем **Audio File**. Откроется диалог выбора файла, отфильтрованный по поддерживаемым аудио- и видеоформатам. Выберите нужный файл и нажмите **Open**. Путь к файлу отобразится в поле.

### 3. Назовите задание

Поле **Job Name** заполняется автоматически на основе имени файла. При необходимости отредактируйте его — это имя будет отображаться в истории транскрипций на главном экране.

### 4. Запустите транскрипцию

Нажмите `Start Transcription`. Приложение перейдёт в режим **Прогресс**.

Чтобы вернуться назад без запуска, нажмите `← Back`.

## Что происходит дальше

Задание выполняется в два этапа, отображаемых на индикаторе прогресса:

1. **Анализ аудио** — диаризация дикторов: определение того, кто говорит и в какой момент.
2. **Распознавание речи** — преобразование речи в текст по сегментам.

Распознанные сегменты появляются в интерактивной таблице по мере их обработки. После завершения обработки приложение автоматически переходит в режим **Результаты**.

Если задание добавлено в момент, когда уже выполняется другое, новое задание получит статус `queued` и начнёт выполнение после завершения текущего. См. [Мониторинг заданий](monitoring_jobs.md).
