# Установка CUDA и cuDNN для ускорения на GPU

Vernacula-Desktop может использовать NVIDIA GPU для значительного ускорения транскрибирования. Для работы с GPU необходимо установить на компьютер набор инструментов NVIDIA CUDA Toolkit и библиотеки среды выполнения cuDNN.

## Требования

- NVIDIA GPU с поддержкой CUDA (рекомендуется GeForce GTX серии 10 или новее).
- Windows 10 или 11 (64-разрядная).
- Файлы моделей должны быть уже загружены. См. раздел [Загрузка моделей](downloading_models.md).

## Шаги установки

### 1. Установите CUDA Toolkit

Скачайте и запустите установщик CUDA Toolkit с сайта для разработчиков NVIDIA. В процессе установки используйте пути по умолчанию. Установщик автоматически задаёт переменную среды `CUDA_PATH` — Vernacula-Desktop использует эту переменную для поиска библиотек CUDA.

### 2. Установите cuDNN

Скачайте ZIP-архив cuDNN для вашей версии CUDA с сайта для разработчиков NVIDIA. Распакуйте архив и скопируйте содержимое папок `bin`, `include` и `lib` в соответствующие папки внутри каталога установки CUDA Toolkit (путь указан в переменной `CUDA_PATH`).

Кроме того, можно установить cuDNN с помощью установщика NVIDIA cuDNN, если он доступен для вашей версии CUDA.

### 3. Перезапустите приложение

После завершения установки закройте и снова откройте Vernacula-Desktop. При запуске приложение проверяет наличие CUDA.

## Статус GPU в настройках

Откройте `Settings…` в строке меню и перейдите в раздел **Hardware & Performance**. Когда компонент обнаружен, рядом с ним отображается галочка (✓):

| Элемент | Значение |
|---|---|
| Название GPU и объём VRAM | Ваш NVIDIA GPU найден |
| CUDA Toolkit ✓ | Библиотеки CUDA найдены через `CUDA_PATH` |
| cuDNN ✓ | Библиотеки DLL среды выполнения cuDNN найдены |
| CUDA Acceleration ✓ | ONNX Runtime загрузил провайдер выполнения CUDA |

Если после установки какой-либо элемент отсутствует, нажмите `Re-check`, чтобы повторно запустить определение оборудования без перезапуска приложения.

В окне настроек также есть прямые ссылки для скачивания CUDA Toolkit и cuDNN, если они ещё не установлены.

### Устранение неполадок

Если рядом с `CUDA Acceleration` не отображается галочка, проверьте следующее:

- Переменная среды `CUDA_PATH` задана (проверьте в разделе `System > Advanced system settings > Environment Variables`).
- Библиотеки DLL cuDNN находятся в каталоге, включённом в системный `PATH`, или в папке `bin` CUDA.
- Драйвер GPU обновлён до последней версии.

### Размер пакета

Когда CUDA активна, в разделе **Hardware & Performance** также отображается текущий динамический предел размера пакета — максимальное количество секунд аудио, обрабатываемых за один запуск GPU. Это значение рассчитывается исходя из свободного объёма VRAM после загрузки моделей и автоматически обновляется при изменении доступного объёма памяти.

## Работа без GPU

Если CUDA недоступна, Vernacula-Desktop автоматически переключается на обработку с помощью CPU. Транскрибирование по-прежнему будет работать, однако медленнее — особенно для длинных аудиофайлов.
