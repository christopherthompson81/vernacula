# Добавление нескольких аудиофайлов в очередь

Используйте **Массовое добавление заданий**, чтобы поставить несколько аудио- или видеофайлов в очередь на транскрибирование за один шаг. Приложение обрабатывает их по одному в порядке добавления.

## Предварительные условия

- Все файлы моделей должны быть загружены. Карточка **Статус модели** должна отображать `All N model file(s) present ✓`. См. раздел [Загрузка моделей](../first_steps/downloading_models.md).

## Как выполнить массовое добавление заданий

1. На главном экране нажмите `Bulk Add Jobs`.
2. Откроется диалог выбора файлов. Выберите один или несколько аудио- или видеофайлов — удерживайте `Ctrl` или `Shift` для выбора нескольких файлов.
3. Нажмите **Открыть**. Каждый выбранный файл будет добавлен в таблицу **История транскрибирования** как отдельное задание.

> **Видеофайлы с несколькими аудиопотоками:** если видеофайл содержит более одного аудиопотока (например, несколько языков или комментарий режиссёра), приложение автоматически создаёт по одному заданию для каждого потока.

## Названия заданий

Каждому заданию автоматически присваивается имя на основе имени соответствующего аудиофайла. Переименовать задание можно в любой момент: нажмите на его название в столбце **Название** таблицы «История транскрибирования», отредактируйте текст и нажмите `Enter` или щёлкните в другом месте.

## Поведение очереди

- Если в данный момент ни одно задание не выполняется, первый файл начинает обрабатываться немедленно, а остальные отображаются со статусом `queued`.
- Если задание уже выполняется, все вновь добавленные файлы отображаются со статусом `queued` и будут запускаться автоматически по очереди.
- Чтобы отслеживать выполнение активного задания, нажмите `Monitor` в столбце **Действия**. См. раздел [Мониторинг заданий](monitoring_jobs.md).
- Чтобы приостановить или удалить задание из очереди до его запуска, воспользуйтесь кнопками `Pause` или `Remove` в столбце **Действия**. См. раздел [Приостановка, возобновление и удаление заданий](pausing_resuming_removing.md).
