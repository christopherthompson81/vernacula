# Налаштування

Вікно **Налаштування** надає вам контроль над конфігурацією обладнання, керуванням моделями, режимом сегментації, поведінкою редактора, зовнішнім виглядом та мовою інтерфейсу. Відкрийте його через рядок меню: `Settings…`.

## Обладнання та продуктивність

У цьому розділі відображається стан вашого GPU NVIDIA та програмного стека CUDA, а також повідомляється про динамічну межу пакетної обробки під час транскрибування на GPU.

| Елемент | Опис |
|---|---|
| Назва GPU та обсяг VRAM | Виявлений GPU NVIDIA та доступна відеопам'ять. |
| CUDA Toolkit | Чи знайдено бібліотеки середовища виконання CUDA через `CUDA_PATH`. |
| cuDNN | Чи доступні DLL-файли середовища виконання cuDNN. |
| Прискорення CUDA | Чи ONNX Runtime успішно завантажив постачальника виконання CUDA. |

Натисніть `Re-check`, щоб повторно виконати виявлення обладнання без перезапуску програми — це корисно після встановлення CUDA або cuDNN.

Прямі посилання для завантаження CUDA Toolkit і cuDNN відображаються у разі, якщо ці компоненти не виявлено.

Повідомлення про **межу пакетної обробки** вказує, скільки секунд аудіо обробляється за один запуск на GPU. Це значення визначається на основі вільної VRAM після завантаження моделей і налаштовується автоматично.

Повні інструкції з налаштування CUDA дивіться у розділі [Встановлення CUDA та cuDNN](cuda_installation.md).

## Моделі

У цьому розділі здійснюється керування файлами моделей штучного інтелекту, необхідними для транскрибування.

- **Точність моделі** — виберіть `INT8 (smaller download)` або `FP32 (more accurate)`. Дивіться [Вибір точності ваг моделі](model_precision.md).
- **Завантажити відсутні моделі** — завантажує файли моделей, яких ще немає на диску. Рядок прогресу та рядок стану відстежують завантаження кожного файлу.
- **Перевірити наявність оновлень** — перевіряє, чи доступні новіші ваги моделей. Банер оновлення також автоматично з'являється на головному екрані, коли виявляються оновлені ваги.

## Режим сегментації

Керує тим, як аудіо розбивається на сегменти перед розпізнаванням мовлення.

| Режим | Опис |
|---|---|
| **Діаризація мовців** | Використовує модель SortFormer для визначення окремих мовців і позначення кожного сегмента. Найкраще підходить для інтерв'ю, нарад і записів із кількома мовцями. |
| **Виявлення голосової активності** | Використовує Silero VAD для виявлення лише фрагментів із мовленням — без позначок мовців. Працює швидше за діаризацію і добре підходить для аудіо з одним мовцем. |

## Редактор транскрипту

**Стандартний режим відтворення** — встановлює режим відтворення, який використовується під час відкриття редактора транскриптів. Ви також можете змінити його безпосередньо в редакторі в будь-який момент. Опис кожного режиму дивіться у розділі [Редагування транскриптів](../operations/editing_transcripts.md).

## Зовнішній вигляд

Виберіть мову відображення інтерфейсу програми. Зміна застосовується негайно. Дивіться [Вибір мови](language.md).
