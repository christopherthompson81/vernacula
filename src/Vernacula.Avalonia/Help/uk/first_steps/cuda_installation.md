# Встановлення CUDA та cuDNN для апаратного прискорення GPU

Parakeet Transcription може використовувати NVIDIA GPU для значного прискорення транскрибування. Апаратне прискорення GPU вимагає встановлення NVIDIA CUDA Toolkit та бібліотек cuDNN на вашому комп'ютері.

## Вимоги

- NVIDIA GPU з підтримкою CUDA (рекомендується GeForce GTX 10-ї серії або новіше).
- Windows 10 або 11 (64-розрядна).
- Файли моделей мають бути вже завантажені. Див. [Завантаження моделей](downloading_models.md).

## Кроки встановлення

### 1. Встановіть CUDA Toolkit

Завантажте та запустіть інсталятор CUDA Toolkit із веб-сайту для розробників NVIDIA. Під час встановлення прийміть шляхи за замовчуванням. Інсталятор автоматично задає змінну середовища `CUDA_PATH` — Parakeet використовує цю змінну для пошуку бібліотек CUDA.

### 2. Встановіть cuDNN

Завантажте ZIP-архів cuDNN для вашої встановленої версії CUDA із веб-сайту для розробників NVIDIA. Розпакуйте архів і скопіюйте вміст його папок `bin`, `include` та `lib` до відповідних папок у каталозі встановлення CUDA Toolkit (шлях, що відображається змінною `CUDA_PATH`).

Як альтернативу, встановіть cuDNN за допомогою інсталятора NVIDIA cuDNN, якщо він доступний для вашої версії CUDA.

### 3. Перезапустіть застосунок

Закрийте та знову відкрийте Parakeet Transcription після встановлення. Застосунок перевіряє наявність CUDA під час запуску.

## Стан GPU у налаштуваннях

Відкрийте `Settings…` у рядку меню та перегляньте розділ **Hardware & Performance**. Кожен компонент відображає позначку (✓), якщо його виявлено:

| Елемент | Що це означає |
|---|---|
| Назва GPU та VRAM | Знайдено ваш NVIDIA GPU |
| CUDA Toolkit ✓ | Бібліотеки CUDA знайдено через `CUDA_PATH` |
| cuDNN ✓ | Знайдено DLL-файли середовища виконання cuDNN |
| CUDA Acceleration ✓ | ONNX Runtime завантажив провайдер виконання CUDA |

Якщо після встановлення якийсь елемент відсутній, натисніть `Re-check`, щоб повторно запустити визначення апаратного забезпечення без перезапуску застосунку.

Вікно налаштувань також містить прямі посилання для завантаження CUDA Toolkit та cuDNN, якщо їх ще не встановлено.

### Усунення несправностей

Якщо `CUDA Acceleration` не відображає позначку, перевірте:

- Чи задано змінну середовища `CUDA_PATH` (перевірте `System > Advanced system settings > Environment Variables`).
- Чи знаходяться DLL-файли cuDNN у каталозі, що включено до системної змінної `PATH`, або в папці `bin` CUDA.
- Чи оновлено драйвер вашого GPU.

### Розмір пакету

Коли CUDA активний, розділ **Hardware & Performance** також відображає поточну динамічну межу пакету — максимальну кількість секунд аудіо, що обробляється за один запуск GPU. Це значення обчислюється на основі вільної пам'яті VRAM після завантаження моделей і автоматично коригується залежно від доступного обсягу пам'яті.

## Робота без GPU

Якщо CUDA недоступний, Parakeet автоматично переходить до обробки на CPU. Транскрибування продовжує працювати, але буде повільнішим, особливо для довгих аудіофайлів.
